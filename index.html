JavaScript Async Topics Blog

1. JavaScript Async

Asynchronous JavaScript refers to the ability to perform tasks without blocking the main thread. Functions can be executed after a delay or once a task is completed.


// Example: Using setTimeout to delay execution
console.log('Start');
setTimeout(() => {
  console.log('Async Task Complete');
}, 2000);
console.log('End');

2. Callback

A callback function is passed as an argument to another function and is executed after the completion of that function.


// Example: Callback function
function fetchData(callback) {
  setTimeout(() => {
    console.log('Data fetched');
    callback();
  }, 1000);
}

function processData() {
  console.log('Processing data');
}

fetchData(processData);

3. Promise

A promise represents a value that may be available now, in the future, or never. It has three states: pending, fulfilled, or rejected.


// Example: Using Promise
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Task completed');
  }, 1500);
});

promise.then(result => console.log(result)).catch(error => console.log(error));

4. Async/Await

Async functions allow for asynchronous code to be written in a more readable manner using await to pause the function until the Promise is resolved.


// Example: Async/Await
async function fetchData() {
  let data = await new Promise((resolve) => {
    setTimeout(() => resolve('Data retrieved'), 2000);
  });
  console.log(data);
}

fetchData();

5. Event Loop

The event loop allows JavaScript to perform non-blocking I/O operations, even though JavaScript is single-threaded.


// Example: Understanding Event Loop
console.log('Start');
setTimeout(() => {
  console.log('Inside timeout');
}, 0);
console.log('End');
